# 통합 주문 작업실 샘플

카카오·쿠팡 주문 엑셀을 하나의 주문표로 통합하고 제작/출고 담당자를 기록하는 웹 샘플입니다.

## 실행

`.xlsx` 파일은 Python 3만으로 처리하며, 구형 `.xls` 파일 지원에는 LibreOffice가 필요합니다.

```bash
python3 src/server.py
```

브라우저에서 `http://localhost:3000`을 엽니다. 같은 네트워크의 다른 PC에서는 서버 PC의 IP와 포트 `3000`으로 접속할 수 있습니다.

## 제공 기능

- 카카오 및 쿠팡 주문 엑셀(`.xlsx`, `.xls`) 여러 개 동시 업로드
- 여러 엑셀이 들어 있는 ZIP 파일 업로드
- 채널·주문·상품 기준 중복 주문 제외
- 통합 주문 목록 검색 및 상태 필터
- 준비 중 선점 체크, 담당자 표시 및 다른 작업자의 중복 선점 방지
- 제품 관리번호(바코드) 입력 및 중복 등록 방지
- 수기 주문 직접 등록 및 주문번호 중복 방지
- 총책임자, 개발자, AS 담당자, 판매 담당자, MD, 일반 작업자 역할 구분
- 로그인 화면에서 일반 작업자 회원가입
- 총책임자·개발자 전용 `회원 관리` 탭에서 가입자 정보·권한·비밀번호·사용 상태 수정 및 삭제
- 총책임자·개발자·판매 담당자·MD는 주문 등록·엑셀 가져오기·출고 엑셀 가능
- AS 담당자와 일반 작업자는 주문 조회 및 작업 상태 처리 가능
- 로그인 계정 기준으로 준비·관리번호·제작·출고 담당자 자동 기록
- 제작 완료 작업자와 처리 시각 기록
- 출고 완료 작업자, 택배사, 송장번호와 처리 시각 기록
- 이전 출고 건과 새 출고 건을 하나의 누적 `.xlsx`로 다운로드
- 새로 내보낸 출고 완료 주문은 일반 주문 목록에서 보관 처리
- 완료 주문 보관함에서 출고 처리된 주문 검색 및 이력 조회
- 판매 담당자 이상 주문 취소, 취소 사유 기록 및 취소 주문 보관함
- 제작 완료 주문 취소는 총책임자·개발자만 가능하며 출고 완료 주문은 취소 불가
- AS 담당자 전용 고객 검색에서 이름·연락처로 고객별 전체 출고 제품과 배송 이력 조회
- `data/orders.json`에 주문 데이터 보존

## 샘플 범위와 운영 전 보완점

이 버전은 흐름 검증용입니다. 작업자 비밀번호는 PBKDF2 해시로 저장되며 관리자만 작업자 계정을 추가할 수 있습니다. 서버 재시작 시에는 보안을 위해 다시 로그인해야 합니다. 실제 외부 운영 전에는 HTTPS, 자동 백업, 로그인 시도 제한, 개인정보 접근 로그, 보존·파기 정책을 추가해야 합니다.

환경변수 `PORT`로 포트를, `DATA_FILE`로 주문 저장 경로를 바꿀 수 있습니다.
